# Contributing to lake-skills

## Branch Naming

- `feat/<skill-name>`
- `fix/<skill-name>`
- `docs/<topic>`

## Commit Message

Use conventional style:

- `feat: add xxx skill`
- `fix: correct xxx trigger rule`
- `docs: update xxx examples`

## Pull Request Checklist

- [ ] Skill has clear "when to use" and "when not to use".
- [ ] Skill includes at least one runnable example.
- [ ] Breaking behavior changes are explained.
- [ ] Docs are updated.

## Review Rules

- At least one approval before merge.
- No direct push to `main`.
- Prefer small, reviewable PRs.
